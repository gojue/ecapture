# 日志与输出配置

## 概述

eCapture 提供灵活的输出配置，支持多种输出目标和日志管理方式，方便与外部系统集成。

## 输出目标

### 标准输出（默认）

不指定任何输出地址时，eCapture 将事件输出到 stdout：

```bash
sudo ecapture tls
```

### 文件输出

```bash
# 事件输出到文件
sudo ecapture tls -l /var/log/ecapture/capture.log
```

### TCP 远程转发

通过 `--logaddr` 参数将日志转发到远程 TCP 服务：

```bash
# 转发到远程日志收集器
sudo ecapture tls --logaddr tcp://192.168.1.100:5514
```

### WebSocket 推送

通过 `--ecaptureq` 参数启用 WebSocket 事件推送，详见 [远程事件流（eCaptureQ）](远程事件流-eCaptureQ)。

## 日志输出优先级

eCapture 按以下优先级选择输出方式：

1. **eCaptureQ EventWriter**（如果配置了 `--ecaptureq`）→ WebSocket 推送
2. **eventAddr 为空或 stdout** → 使用 Logger Writer 输出到终端
3. **其他 eventAddr** → 通过 Writer 工厂创建对应的输出器（文件/TCP/WebSocket）

## 日志文件轮转

eCapture 支持基于时间和大小的日志轮转（rotatelog）：

| 参数 | 说明 |
|------|------|
| `--eventroratesize` | 按文件大小轮转（单位 MB） |
| `--eventroratetime` | 按时间轮转（单位小时） |

```bash
# 每 100MB 轮转一次
sudo ecapture tls -l /var/log/ecapture/capture.log --eventroratesize=100

# 每 24 小时轮转一次
sudo ecapture tls -l /var/log/ecapture/capture.log --eventroratetime=24
```

## Writer 工厂机制

eCapture 使用 Writer 工厂模式（`internal/output/writers/factory.go`）根据地址格式自动创建对应的 Writer：

| 地址格式 | Writer | 说明 |
|---------|--------|------|
| 空 / `stdout` | LoggerWriter | 通过 zerolog 输出到终端 |
| 文件路径 | FileWriter | 输出到本地文件 |
| `tcp://host:port` | TcpWriter | TCP 远程转发 |
| `ws://host:port` | WebSocketWriter | WebSocket 推送 |

## 与外部系统对接

### ELK Stack

通过 TCP Writer 将日志转发到 Logstash：

```bash
# Logstash 配置一个 TCP input
sudo ecapture tls --logaddr tcp://logstash-host:5044
```

### Kafka

可以通过中间代理（如 Filebeat / Fluentd）将 eCapture 的文件输出转发到 Kafka：

```bash
# eCapture 输出到文件
sudo ecapture tls -l /var/log/ecapture/output.log

# Filebeat 配置监控该文件并转发到 Kafka
```

### SIEM 系统

通过 TCP 或文件输出集成到 Splunk / QRadar 等 SIEM 系统。
