# 整体三层架构

eCapture 采用三层架构设计，数据从内核挂钩点流向最终输出。

## 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                   CLI 与输出层                            │
│  cobra CLI → factory → probe → dispatcher → writer       │
│  HTTP API（运行时配置更新）                                │
├─────────────────────────────────────────────────────────┤
│                   用户态探针层                             │
│  BaseProbe → OpenSSL/GoTLS/GnuTLS/NSPR/Bash/MySQL/...   │
│  EventProcessor → IWorker → IParser                      │
├─────────────────────────────────────────────────────────┤
│                   eBPF 内核层                             │
│  uprobe/kprobe/TC classifier → perf/ring buffer          │
│  C 程序编译为 .o 字节码，CO-RE + 非 CO-RE 双模式          │
└─────────────────────────────────────────────────────────┘
```

## 第一层：eBPF 内核层

### 职责

- 在目标函数（如 `SSL_read`、`SSL_write`）上挂载 uprobe
- 在函数入口/返回处读取参数和返回值中的明文数据
- 通过 perf buffer 或 ring buffer 将数据传递到用户态

### 关键组件

| 组件 | 路径 | 说明 |
|------|------|------|
| eBPF C 源码 | `kern/*.c` | 每个库版本一个文件（如 `openssl_3_0_0_kern.c`） |
| 公共头文件 | `kern/common.h`, `kern/ecapture.h` | 共享的数据结构和宏定义 |
| Master Key 提取 | `kern/openssl_masterkey*.h` | TLS 密钥提取逻辑 |
| TC 程序 | `kern/tc.h` | Traffic Control classifier，用于 pcapng 模式抓包 |
| Go 参数处理 | `kern/go_argument.h` | 处理 Go 的 Register ABI / Stack ABI 差异 |

### CO-RE 与非 CO-RE 的加载差异

| 特性 | CO-RE 模式 | 非 CO-RE 模式 |
|------|-----------|-------------|
| BTF 依赖 | 需要 `/sys/kernel/btf/vmlinux` | 不需要 |
| 编译产物 | `*_kern_core.o` | `*_kern_noncore.o` |
| 可移植性 | 跨内核版本 | 仅限编译时的内核 |
| 构建命令 | `make` | `make nocore` |
| 加载逻辑 | `internal/builder/` | `internal/builder/` |

### 字节码嵌入

编译后的 `.o` 文件通过 `go-bindata` 工具嵌入到 Go 二进制中：

- **生产模式**：`assets/ebpf_probe.go`（build tag `ebpfassets`）—— 由 `make assets` 生成
- **开发模式**：`assets/ebpf_probe_stub.go`（build tag `!ebpfassets`）—— 存根文件

## 第二层：用户态探针层

### 职责

- 加载和管理 eBPF 字节码
- 挂载/卸载 uprobe
- 从 perf/ring buffer 读取事件
- 事件解码和分发

### 关键组件

| 组件 | 路径 | 说明 |
|------|------|------|
| Domain 接口 | `internal/domain/` | `Probe`、`Event`、`Configuration`、`EventDecoder` 接口 |
| BaseProbe | `internal/probe/base/` | 模板方法基类，管理探针生命周期 |
| 各探针实现 | `internal/probe/{openssl,gotls,...}/` | 继承 BaseProbe 的具体探针 |
| 工厂 | `internal/factory/` | 探针注册与实例化 |
| 事件处理器 | `pkg/event_processor/` | UUID 亲和性调度、协议解析 |
| eBPF 加载 | `internal/builder/` | CO-RE / 非 CO-RE 字节码加载逻辑 |

### 数据流路径

```
kern/*.c → 编译 → bytecode/*.o → go-bindata → assets/ebpf_probe.go
                                                      ↓
                              internal/probe/*（加载字节码、挂载 uprobe）
                                                      ↓
                              perf/ring buffer 读取事件
                                                      ↓
                              pkg/event_processor/（协议解析）
                                                      ↓
                              internal/events/（事件分发）
                                                      ↓
                              internal/output/（编码、写入输出）
```

## 第三层：CLI 与输出层

### 职责

- 解析用户命令行参数
- 配置和启动探针
- 编码和输出捕获的数据
- 提供运行时 HTTP 配置接口

### 关键组件

| 组件 | 路径 | 说明 |
|------|------|------|
| Cobra CLI | `cli/cmd/` | 每个子命令一个文件（`tls.go`、`bash.go` 等） |
| HTTP API | `cli/http/` | 运行时配置更新接口 |
| Writers | `internal/output/writers/` | 文件、pcap、keylog、TCP、WebSocket 输出 |
| Encoders | `internal/output/encoders/` | plain、JSON、protobuf 编码 |
| eCaptureQ | `pkg/ecaptureq/` | WebSocket 事件推送服务 |

### Writer + Encoder 组合

输出层采用 Writer（输出目标）+ Encoder（编码格式）的组合模式：

```
EventHandler → Encoder（plain/JSON/protobuf） → Writer（file/TCP/WebSocket/stdout）
```

这种设计允许灵活组合不同的编码格式和输出目标。
