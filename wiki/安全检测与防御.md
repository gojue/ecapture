# 安全检测与防御

eCapture 是一款强大的安全审计工具。本页面为安全团队提供如何检测未经授权运行的 eCapture（或类似 eBPF 工具）以及如何实施防御措施的指导。

## 检测 eBPF 捕获工具

### 1. 检查活跃的 eBPF 程序

```bash
# 列出所有已加载的 eBPF 程序（需要 root）
sudo bpftool prog list

# 查找 uprobe 类型的程序
sudo bpftool prog list | grep -i uprobe
```

eCapture 会加载 uprobe 程序挂钩到 `SSL_read`、`SSL_write`、`SSL_do_handshake` 等函数。任何意外的 uprobe 程序都应调查。

### 2. 检查 Uprobe 事件

```bash
# 查看已注册的 uprobe 事件
sudo cat /sys/kernel/debug/tracing/uprobe_events

# 过滤 SSL/TLS 相关的探针
sudo cat /sys/kernel/debug/tracing/uprobe_events | grep -E "ssl|SSL|gnutls|nspr"
```

### 3. 检查活跃的 Perf Event Buffer

```bash
# 列出 eBPF 使用的 perf event 数组
sudo bpftool map list | grep -i perf
```

### 4. 监控进程执行

```bash
# 检查是否有运行中的 eCapture 进程
ps aux | grep ecapture

# 使用 auditd 监控 eCapture 的执行
sudo auditctl -w /usr/local/bin/ecapture -p x -k ecapture_exec

# 监控 bpf() 系统调用（捕获任何 eBPF 工具）
sudo auditctl -a always,exit -F arch=b64 -S bpf -k bpf_activity
```

### 5. 检查 Docker 容器

```bash
# 列出特权容器
docker ps --format '{{.Names}}' | xargs -I {} docker inspect \
  --format='{{.Name}}: Privileged={{.HostConfig.Privileged}}' {}
```

## 防御策略

### 1. 限制 eBPF 访问

```bash
# 禁止非特权用户使用 BPF
sudo sysctl -w kernel.unprivileged_bpf_disabled=1

# 持久化配置
echo "kernel.unprivileged_bpf_disabled=1" | sudo tee /etc/sysctl.d/99-disable-bpf.conf
```

### 2. 使用 Linux 安全模块

#### AppArmor

```bash
cat << 'EOF' > /etc/apparmor.d/deny-bpf
#include <tunables/global>
profile deny-bpf flags=(attach_disconnected) {
  deny capability sys_admin,
  deny capability bpf,
  deny capability perfmon,
}
EOF
```

#### SELinux

```bash
# 审计 BPF 相关的 AVC 拒绝记录
ausearch -m AVC -ts today | grep bpf
```

### 3. 使用 auditd 监控

```bash
cat << 'EOF' > /etc/audit/rules.d/ebpf-monitor.rules
# 监控 bpf() 系统调用
-a always,exit -F arch=b64 -S bpf -k ebpf_usage

# 监控 perf_event_open
-a always,exit -F arch=b64 -S perf_event_open -k perf_event

# 监控 tracing 文件系统访问
-w /sys/kernel/debug/tracing/ -p rwa -k tracing_access

# 监控 uprobe 注册
-w /sys/kernel/debug/tracing/uprobe_events -p wa -k uprobe_modification
EOF

sudo augenrules --load
```

### 4. Docker 安全加固

生产环境中避免使用 `--privileged=true`，改用最小权限配置：

```bash
docker run --rm \
  --cap-add=BPF --cap-add=PERFMON --cap-add=NET_ADMIN --cap-add=SYS_PTRACE \
  --pid=host --net=host \
  -v /sys/kernel/debug:/sys/kernel/debug:ro \
  -v /sys/fs/bpf:/sys/fs/bpf \
  gojue/ecapture:latest tls
```

### 5. 网络层面检测

如果 eCapture 配置了网络事件转发（如 eCaptureQ），可监控：

```bash
# 检查 eCapture 默认监听端口
ss -tlnp | grep 28256
```

## 合规使用指南

### 安全团队

1. **授权**：在生产环境部署 eCapture 前，必须获得书面授权
2. **范围**：使用 `--pid` 参数限制捕获范围
3. **数据处理**：捕获的明文可能包含敏感数据（密码、令牌、PII），需按数据分类政策处理
4. **审计追踪**：记录 eCapture 的使用时间、地点、操作人员和目的
5. **时效限制**：审计/调试完成后及时停止 eCapture

### 组织层面

1. **政策**：将 eBPF 工具纳入安全工具治理政策
2. **培训**：确保操作人员了解数据敏感性
3. **监控**：部署上述检测机制
