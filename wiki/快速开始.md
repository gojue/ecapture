# 快速开始（5 分钟跑通第一个示例）

本页以最简单的 TLS 明文捕获为例，手把手演示从启动 eCapture 到看到明文输出的完整流程。

## 第一步：确认环境

```bash
# 检查内核版本（x86_64 需要 ≥4.18，aarch64 需要 ≥5.5）
uname -r

# 检查 BTF 支持（CO-RE 模式需要）
ls /sys/kernel/btf/vmlinux
```

## 第二步：启动 eCapture

```bash
# 最简形式：自动发现 OpenSSL 库，text 模式输出
sudo ecapture tls
```

启动后你会看到类似输出：
```
INF AppName="eCapture(旁观者)"
INF Repository=https://github.com/gojue/ecapture
INF Kernel Info=5.15.152 Pid=233698
INF BTF bytecode mode: CORE. btfMode=0
INF module started successfully. moduleName=EBPFProbeOPENSSL
```

## 第三步：触发 HTTPS 请求

在另一个终端执行：

```bash
curl https://baidu.com
```

## 第四步：查看捕获的明文

回到 eCapture 终端，你将看到捕获的 HTTP 请求和响应明文：

```
??? UUID:233479_233479_curl_5_1_39.156.66.10:443, Name:HTTPRequest, Type:1, Length:73
GET / HTTP/1.1
Host: baidu.com
Accept: */*
User-Agent: curl/7.81.0
```

恭喜！你已成功使用 eCapture 捕获了 TLS 加密通讯的明文内容。

## 进阶示例

### 按 PID 过滤

只捕获特定进程的 TLS 通讯：

```bash
# 先获取目标进程 PID
pidof nginx

# 指定 PID 捕获
sudo ecapture tls --pid=12345
```

### 指定 OpenSSL 库路径

当目标程序使用非标准路径的 OpenSSL 时：

```bash
sudo ecapture tls --libssl=/path/to/libssl.so.3
```

### 捕获静态编译程序

对于静态编译的目标程序，直接指定程序路径：

```bash
sudo ecapture tls --libssl=/usr/local/bin/my_app
```

### 针对 curl 捕获

```bash
sudo ecapture tls --pid=$(pidof curl)
```

### 针对 nginx 捕获

```bash
sudo ecapture tls --pid=$(pidof nginx)
```

### 针对 wget 捕获

```bash
# wget 通常使用 GnuTLS
sudo ecapture gnutls
```

## 输出说明

- **UUID** 格式：`PID_TID_进程名_FD_方向_对端地址`
  - 方向：`0` = 读取（接收），`1` = 写入（发送）
- **Name**：事件类型（如 `HTTPRequest`、`HTTP2Response`）
- **Type**：事件类型编号
- **Length**：数据长度

更多输出模式详见 [输出格式说明](输出格式说明)。
