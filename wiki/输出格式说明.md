# 输出格式说明（text / pcap / keylog）

eCapture 支持三种捕获输出模式，以及多种编码器和输出目标的组合。

## 三种捕获模式

### 1. Text 模式（默认）

直接输出捕获的明文数据到终端或指定文件。

```bash
# 输出到终端
sudo ecapture tls -m text

# 输出到文件
sudo ecapture tls -m text -l /tmp/capture.log
```

**适用场景**：快速查看、实时监控、简单审计。

### 2. PcapNG 模式

将捕获的明文数据以 pcap-NG 格式存储，支持 TLS 加密的 HTTP 1.0/1.1/2.0（TCP）及 HTTP/3 QUIC（UDP）协议。配合 Wireshark 可直接查看解密后的网络包。

```bash
# 基本用法
sudo ecapture tls -m pcap -i eth0 --pcapfile=ecapture.pcapng tcp port 443

# 指定网络接口和过滤规则
sudo ecapture tls -m pcap -i ens160 -w ecap.pcapng host 192.168.1.1 and tcp port 443
```

**参数说明**：
- `-m pcap` 或 `-m pcapng`：启用 pcapng 模式
- `-i`：指定网络接口（TC Classifier 挂载点）
- `--pcapfile` 或 `-w`：输出文件路径（默认 `ecapture_openssl.pcapng`）
- 末尾的过滤表达式：BPF 过滤语法（与 tcpdump 类似）

**额外权限**：pcapng 模式需要 `CAP_NET_ADMIN`（用于 TC 挂载）。

**与 Wireshark 配合**：
1. 运行 eCapture 生成 pcapng 文件
2. 用 Wireshark 打开文件即可直接查看明文内容

### 3. Keylog 模式

保存 TLS 握手密钥（Master Secret）到文件，格式为 NSS Key Log Format。配合 tcpdump 抓包和 Wireshark 可以解密 TLS 流量。

```bash
# 保存密钥
sudo ecapture tls -m keylog --keylogfile=openssl_keylog.log

# 配合 tshark 实时解密
tshark -o tls.keylog_file:ecapture_masterkey.log -Y http -T fields -e http.file_data -f "port 443" -i eth0
```

**参数说明**：
- `-m keylog` 或 `-m key`：启用 keylog 模式
- `--keylogfile` 或 `-k`：密钥文件路径（默认 `ecapture_openssl_key.log`）

**与 Wireshark 配合**：
1. 同时运行 eCapture（keylog 模式）和 tcpdump
2. Wireshark 打开抓包文件 → 编辑 → 首选项 → Protocols → TLS → 设置 `(Pre)-Master-Secret log filename`
3. 即可查看解密后的 TLS 流量

## 编码器（Encoder）

eCapture 输出数据支持以下编码格式：

| 编码器 | 说明 | 实现文件 |
|--------|------|---------|
| Plain | 纯文本格式（默认） | `internal/output/encoders/plain_encoder.go` |
| JSON | JSON 结构化输出 | `internal/output/encoders/json_encoder.go` |
| Protobuf | Protocol Buffers 编码 | `internal/output/encoders/protobuf_encoder.go` |

## 输出目标（Writer）

| Writer | 说明 | 触发方式 | 实现文件 |
|--------|------|---------|---------|
| Stdout | 标准输出（默认） | 不指定输出地址 | `stdout_writer.go` |
| File | 文件输出 | `-l /path/to/file` | `file_writer.go` |
| Pcap | pcapng 文件 | `-m pcap` | `pcap_writer.go` |
| Keylog | NSS Key Log 文件 | `-m keylog` | `keylog_writer.go` |
| TCP | TCP 远程转发 | `--logaddr tcp://host:port` | `tcp_writer.go` |
| WebSocket | WebSocket 推送 | `--ecaptureq` 模式 | `websocket_writer.go` |

Writer 工厂通过地址格式自动选择合适的 Writer，注册逻辑在 `internal/output/writers/factory.go`。

## 组合使用示例

```bash
# text 模式 + 文件输出
sudo ecapture tls -m text -l /tmp/tls_capture.log

# pcapng 模式 + 指定接口
sudo ecapture tls -m pcap -i wlan0 --pcapfile=/tmp/capture.pcapng tcp port 443

# keylog 模式 + tshark 实时解密
sudo ecapture tls -m keylog -k /tmp/keys.log &
tshark -o tls.keylog_file:/tmp/keys.log -Y http -i eth0
```
