# 事件处理流水线

## 概述

事件处理流水线负责将 eBPF perf/ring buffer 中的原始事件数据解析为结构化的协议事件（如 HTTP 请求/响应），并路由到正确的输出端。

## 核心组件

### EventProcessor

`pkg/event_processor/processor.go` 是事件调度中枢：

```go
type EventProcessor struct {
    incoming    chan IEventStruct    // 接收来自探针的新事件
    outComing   chan []byte          // 输出到 Writer
    destroyConn chan uint64          // 连接销毁通知
    workerQueue map[string]IWorker  // UUID → Worker 映射
}
```

**调度循环**（`Serve()` 方法）：

1. 从 `incoming` channel 读取新事件
2. 调用 `dispatch()` 将事件路由到对应的 Worker
3. 处理 Worker 销毁和输出写入

### IWorker（eventWorker）

`pkg/event_processor/iworker.go` 定义了事件处理 Worker：

- **UUID 亲和性**：每个 Worker 绑定一个唯一 UUID（格式：`PID_TID_Comm_FD_方向_对端地址`）
- **保证有序处理**：同一连接的事件始终由同一个 Worker 处理
- **协议解析**：Worker 内部调用 IParser 进行协议识别和解析

### UUID 生成规则

```
sock:PID_TID_CommName_FD_Direction_RemoteAddr
```

- **PID**：进程 ID
- **TID**：线程 ID
- **CommName**：进程名
- **FD**：文件描述符
- **Direction**：0（读取）/ 1（写入）
- **RemoteAddr**：对端地址（IP:Port）

## 协议解析器（IParser）

`pkg/event_processor/iparser.go` 定义了协议解析接口：

### 支持的解析器

| 解析器 | 说明 | 实现文件 |
|--------|------|---------|
| HTTP/1.1 Request | HTTP/1.1 请求解析 | `http_request.go` |
| HTTP/1.1 Response | HTTP/1.1 响应解析 | `http_response.go` |
| HTTP/2 Request | HTTP/2 帧解析（HPACK 头部解码） | `http2_request.go` |
| HTTP/2 Response | HTTP/2 帧解析 | `http2_response.go` |
| Default | 默认处理（原始数据透传） | 内置 |

### 协议识别流程

1. 读取数据的前几个字节
2. 尝试匹配已知协议签名：
   - HTTP/2：`PRI * HTTP/2.0\r\n` 前缀或 HTTP/2 帧头
   - HTTP/1.x：`GET`、`POST`、`HTTP/1.` 等关键字
3. 匹配成功则使用对应的 Parser
4. 无法匹配则使用 DefaultParser（原始数据输出）

### ProcessStatus 状态机

Worker 内部维护处理状态：

```
Init → Processing → Done → (回收)
                ↓
            Timeout → (强制回收)
```

- **Init**：等待第一个数据包
- **Processing**：正在积累和解析数据
- **Done**：协议消息完整，可以输出
- **Timeout**：超时回收（由 ticker 触发）

## Ticker 超时回收

EventProcessor 内部运行一个定时器（ticker），定期检查 Worker 的活跃状态：

- 长时间没有新事件的 Worker 会被标记为超时
- 超时的 Worker 会输出已积累的数据并释放资源
- 这确保了连接中断时数据不会永久滞留在内存中

## 完整数据流

```
eBPF Perf Buffer
       ↓ perf_event_read
BaseProbe.GoReaderLoop()
       ↓ 解码事件
EventDecoder.Decode()
       ↓ 事件对象
EventDispatcher.Dispatch()
       ↓ 分发到 Handler
EventHandler.Handle()
       ↓ (对于 Processor 类型事件)
EventProcessor.incoming channel
       ↓ dispatch()
IWorker (按 UUID 亲和性路由)
       ↓ 协议解析
IParser (HTTP/1.1, HTTP/2, Default)
       ↓ 输出
EventProcessor.outComing channel
       ↓
Writer (file/pcap/keylog/TCP/WebSocket/stdout)
```
