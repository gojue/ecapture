# 安装与环境要求

## 安装方式

### 方式一：预编译二进制包（推荐）

从 [GitHub Releases](https://github.com/gojue/ecapture/releases) 下载对应架构的预编译包：

```bash
# 下载最新版本（以 x86_64 为例）
wget https://github.com/gojue/ecapture/releases/latest/download/ecapture-linux-amd64.tar.gz

# 解压
tar -xzf ecapture-linux-amd64.tar.gz

# 运行
sudo ./ecapture --help
```

支持的包格式：
- **tar.gz**：通用 Linux 二进制包
- **RPM**：适用于 CentOS / RHEL / Fedora
- **DEB**：适用于 Ubuntu / Debian

### 方式二：Docker 镜像

```bash
# 拉取镜像
docker pull gojue/ecapture:latest

# 运行（需要特权模式或指定 Capabilities）
docker run --rm --privileged=true --net=host -v ${HOST_PATH}:${CONTAINER_PATH} gojue/ecapture ARGS
```

> **安全提醒**：生产环境中建议使用细粒度的 Capabilities 替代 `--privileged=true`，详见 [最小权限配置](最小权限配置)。

### 方式三：从源码编译

详见 [编译与构建](编译与构建) 页面。快速步骤：

```bash
git clone --recurse-submodules https://github.com/gojue/ecapture.git
cd ecapture
make          # CO-RE 模式（推荐，需要 BTF 支持）
# 或
make nocore   # 非 CO-RE 模式（不需要 BTF）
```

## 前置依赖

### 运行时依赖

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Linux 内核 | x86_64 ≥4.18, aarch64 ≥5.5 | eBPF 支持 |
| ROOT 权限 | - | 或通过 setcap 设置 Capabilities |
| BTF 支持 | 可选 | CO-RE 模式需要，`ls /sys/kernel/btf/vmlinux` |

### 编译时依赖（仅从源码编译时需要）

| 工具 | 版本要求 |
|------|---------|
| Go | ≥1.24 |
| Clang | ≥9 |
| LLVM (llc) | ≥9 |
| cmake | ≥3.18.4 |
| bpftool | - |
| pkgconf | - |
| libelf-dev | - |

> Ubuntu 20.04+ 可使用一键初始化脚本：
> ```bash
> /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/gojue/ecapture/master/builder/init_env.sh)"
> ```

## 常见安装问题

### Q：提示 "kernel version not supported"
**A**：检查内核版本是否满足最低要求：
```bash
uname -r
```

### Q：提示 "BTF is not available"
**A**：当前内核未启用 BTF。可使用非 CO-RE 版本，或升级内核。检查方法：
```bash
ls /sys/kernel/btf/vmlinux
```

### Q：Docker 容器内运行失败
**A**：确保使用了 `--privileged=true` 或足够的 `--cap-add` 参数，并且挂载了必要的文件系统路径。详见 [最小权限配置](最小权限配置)。
