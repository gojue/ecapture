# 探针框架与扩展机制

## 领域接口设计

eCapture 在 `internal/domain/` 目录定义了核心接口，确保各层之间的解耦。

### Probe 接口

```go
type Probe interface {
    Initialize(ctx context.Context, config Configuration) error
    Start(ctx context.Context) error
    Stop(ctx context.Context) error
    Close() error
    Name() string
    IsRunning() bool
    Events() []*ebpf.Map
}
```

**生命周期**：`Initialize` → `Start` → `Stop` → `Close`

### Configuration 接口

```go
type Configuration interface {
    Validate() error
    GetPid() uint64
    GetUid() uint64
    GetDebug() bool
    GetHex() bool
    GetBTF() uint8
    GetPerCpuMapSize() int
    GetTruncateSize() uint64
    EnableGlobalVar() bool
    GetByteCodeFileMode() uint8
    Bytes() []byte
    // ... 其他方法
}
```

### Event 接口

```go
type Event interface {
    DecodeFromBytes(data []byte) error
    String() string
    StringHex() string
    Clone() Event
    Type() EventType
    UUID() string
    Validate() error
}
```

### EventDecoder 接口

```go
type EventDecoder interface {
    Decode(em *ebpf.Map, data []byte) (Event, error)
    GetDecoder(em *ebpf.Map) (Event, bool)
}
```

### EventDispatcher 接口

```go
type EventDispatcher interface {
    Register(handler EventHandler) error
    Unregister(handlerName string) error
    Dispatch(event Event) error
    Close() error
}
```

## BaseProbe 模板方法模式

`internal/probe/base/BaseProbe` 是所有探针的基类，采用模板方法模式：

### 基类提供的功能

1. **配置验证**：调用 `Configuration.Validate()` 验证配置
2. **Logger 初始化**：创建带探针名称的日志器
3. **EventDispatcher 创建**：初始化事件分发器
4. **Writer 选择**：根据配置优先级选择输出 Writer（eCaptureQ → Logger → 文件/TCP/WebSocket）
5. **TextHandler 注册**：注册文本处理器到 Dispatcher
6. **生命周期管理**：`isRunning` 状态管理、资源释放

### 具体探针需要实现的逻辑

具体探针（如 `internal/probe/openssl/`）嵌入 `BaseProbe`，只需实现：

1. **配置类**（`config.go`）：嵌入 `BaseConfig`，添加探针特有字段和验证
2. **事件类**（`event.go`）：实现 `Event` 接口的 `DecodeFromBytes()` 和 `Validate()`
3. **探针逻辑**（`*_probe.go`）：实现 `Initialize()`、`Start()`、`Stop()`、`Close()`
4. **工厂注册**（`register.go`）：通过 `init()` 函数注册到全局工厂

## 工厂注册模式

### 全局工厂

`internal/factory/probe_factory.go` 实现了探针工厂：

```go
// 工厂接口
type ProbeFactory interface {
    CreateProbe(probeType ProbeType) (domain.Probe, error)
    RegisterProbeConstructor(probeType ProbeType, constructor ProbeConstructor) error
    GetSupportedProbes() []ProbeType
}

// 全局工厂实例
var globalFactory = NewProbeFactory()
```

### 探针自注册

每个探针在 `register.go` 中通过 `init()` 函数自动注册：

```go
// internal/probe/openssl/register.go
func init() {
    factory.RegisterProbe(factory.ProbeTypeOpenSSL, func() (domain.Probe, error) {
        return NewOpenSSLProbe()
    })
}
```

### 已注册的探针类型

| ProbeType | 值 | 探针 |
|-----------|-----|------|
| `ProbeTypeBash` | "Bash" | Bash 命令捕获 |
| `ProbeTypeZsh` | "Zsh" | Zsh 命令捕获 |
| `ProbeTypeMySQL` | "MySQL" | MySQL SQL 审计 |
| `ProbeTypePostgres` | "postgres" | PostgreSQL SQL 审计 |
| `ProbeTypeOpenSSL` | "OpenSSL" | OpenSSL/BoringSSL TLS 捕获 |
| `ProbeTypeGnuTLS` | "GnuTLS" | GnuTLS TLS 捕获 |
| `ProbeTypeNSPR` | "NSPR" | NSS/NSPR TLS 捕获 |
| `ProbeTypeGoTLS` | "GoTLS" | Go TLS 捕获 |

## BaseConfig 配置继承

`internal/config/` 目录下的 `BaseConfig` 提供通用配置：

- PID / UID 过滤
- Debug / Hex 模式
- BTF 模式选择
- PerCpuMapSize
- TruncateSize
- Logger / Event 输出地址

具体探针的配置类嵌入 `BaseConfig` 并添加特有字段（如 OpenSSL 的 `LibsslPath`、`CaptureMode` 等）。

## 接口职责边界

```
domain.Configuration ← 配置验证与参数获取
        ↓
domain.Probe ← 探针生命周期管理
        ↓
domain.EventDecoder ← eBPF Map 数据解码
        ↓
domain.Event ← 事件数据结构
        ↓
domain.EventDispatcher ← 事件分发到多个 Handler
        ↓
domain.EventHandler ← 事件处理（写入 Writer）
```
