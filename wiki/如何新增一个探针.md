# 如何新增一个探针

本页以 Bash 探针为参考，手把手指导开发者为 eCapture 添加新探针。

## 步骤总览

1. 创建 `internal/probe/<name>/config.go` — 配置
2. 创建 `internal/probe/<name>/event.go` — 事件定义
3. 创建 `internal/probe/<name>/<name>_probe.go` — 探针实现
4. 创建 `internal/probe/<name>/register.go` — 工厂注册
5. 添加 eBPF C 源码 `kern/<name>_kern.c`
6. 在 `variables.mk` 的 `TARGETS` 列表中添加新目标
7. 添加 CLI 子命令 `cli/cmd/<name>.go`
8. 编写单元测试

## 第一步：配置（config.go）

嵌入 `config.BaseConfig`，添加探针特有字段：

```go
// internal/probe/<name>/config.go
package name

import "github.com/gojue/ecapture/internal/config"

type Config struct {
    config.BaseConfig
    // 探针特有字段
    TargetPath string `json:"target_path"`
}

func NewConfig() *Config {
    cfg := &Config{}
    cfg.SetPerCpuMapSize(4096)
    return cfg
}

func (c *Config) Validate() error {
    if err := c.BaseConfig.Validate(); err != nil {
        return err
    }
    // 探针特有验证逻辑
    return nil
}
```

**参考**：`internal/probe/bash/config.go`

## 第三步：探针实现（probe.go）

嵌入 `base.BaseProbe`，实现生命周期方法：

```go
// internal/probe/<name>/<name>_probe.go
package name

import (
    "context"
    "github.com/gojue/ecapture/internal/probe/base"
)

type Probe struct {
    *base.BaseProbe
    config *Config
}

func NewProbe() (*Probe, error) {
    return &Probe{
        BaseProbe: base.NewBaseProbe("EBPFProbe<NAME>"),
    }, nil
}

func (p *Probe) Initialize(ctx context.Context, cfg domain.Configuration) error {
    // 1. 调用基类初始化
    if err := p.BaseProbe.Initialize(ctx, cfg); err != nil {
        return err
    }
    // 2. 探针特有初始化（查找目标库路径等）
    return nil
}

func (p *Probe) Start(ctx context.Context) error {
    // 1. 加载 eBPF 字节码
    // 2. 挂载 uprobe
    // 3. 启动事件读取循环
    return p.BaseProbe.Start(ctx)
}

func (p *Probe) Stop(ctx context.Context) error {
    return p.BaseProbe.Stop(ctx)
}

func (p *Probe) Close() error {
    return p.BaseProbe.Close()
}
```

**参考**：`internal/probe/bash/bash_probe.go`

## 第四步：工厂注册（register.go）

通过 `init()` 函数自动注册到全局工厂：

```go
// internal/probe/<name>/register.go
package name

import "github.com/gojue/ecapture/internal/factory"

func init() {
    _ = factory.RegisterProbe(factory.ProbeType<Name>, func() (domain.Probe, error) {
        return NewProbe()
    })
}
```

> **注意**：需要在 `internal/factory/probe_factory.go` 中添加新的 `ProbeType` 常量。

**参考**：`internal/probe/bash/register.go`

## 第五步：eBPF C 程序

在 `kern/` 目录下创建 eBPF 内核程序：

```c
// kern/<name>_kern.c
#include "ecapture.h"

// 定义事件数据结构（与 Go 的 Event 结构体对齐）
struct event_t {
    u32 pid;
    u32 uid;
    char data[256];
    s32 data_len;
};

// 定义 perf event map
struct {
    __uint(type, BPF_MAP_TYPE_PERF_EVENT_ARRAY);
} events SEC(".maps");

// uprobe 入口函数
SEC("uprobe/target_function")
int probe_entry(struct pt_regs *ctx) {
    struct event_t event = {};
    event.pid = bpf_get_current_pid_tgid() >> 32;
    event.uid = bpf_get_current_uid_gid();
    // ... 读取函数参数
    bpf_perf_event_output(ctx, &events, BPF_F_CURRENT_CPU, &event, sizeof(event));
    return 0;
}

char _license[] SEC("license") = "GPL";
```

**参考**：`kern/bash_kern.c`

## 第六步：更新构建系统

在 `variables.mk` 的 `TARGETS` 列表中添加新目标名称，使 Makefile 能够编译新的 eBPF 程序。

## 第七步：CLI 子命令

在 `cli/cmd/` 目录下添加子命令文件：

```go
// cli/cmd/<name>.go
package cmd

import (
    "github.com/spf13/cobra"
    "github.com/gojue/ecapture/internal/factory"
    nameProbe "github.com/gojue/ecapture/internal/probe/<name>"
)

var nameConfig = nameProbe.NewConfig()

var nameCmd = &cobra.Command{
    Use:   "<name>",
    Short: "capture <name> events",
    RunE:  nameCommandFunc,
}

func init() {
    nameCmd.PersistentFlags().StringVar(&nameConfig.TargetPath, "target", "", "target path")
    rootCmd.AddCommand(nameCmd)
}

func nameCommandFunc(command *cobra.Command, args []string) error {
    nameConfig.SetPid(globalConf.Pid)
    nameConfig.SetUid(globalConf.Uid)
    nameConfig.SetDebug(globalConf.Debug)
    nameConfig.SetHex(globalConf.IsHex)
    nameConfig.SetBTF(globalConf.BtfMode)
    nameConfig.SetPerCpuMapSize(globalConf.PerCpuMapSize)
    nameConfig.SetTruncateSize(globalConf.TruncateSize)
    return runProbe(factory.ProbeType<Name>, nameConfig)
}
```

**参考**：`cli/cmd/bash.go`

## 第八步：单元测试

在探针目录下添加 `_test.go` 文件，测试配置验证和事件解码。

## 注意事项

1. **Build Tags**：eBPF 资产代码使用 `//go:build ebpfassets`，存根使用 `//go:build !ebpfassets`
2. **错误处理**：使用 `internal/errors` 包（结构化错误码），不使用 `fmt.Errorf`
3. **日志**：使用 `internal/logger`（封装 zerolog），不使用 `fmt.Println`
4. **Map 命名**：C 代码中的 eBPF map 名称需要与 Go 代码中 `mapNameToDecoder` 的注册对应
5. **平台限制**：Shell 脚本和 Makefile 必须检查 Linux 和内核版本
