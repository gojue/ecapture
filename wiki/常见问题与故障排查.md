# 常见问题与故障排查

## 安装相关

### Q：提示 "kernel version not supported"

**原因**：当前内核版本低于 eCapture 的最低要求。

**解决**：
```bash
# 检查内核版本
uname -r
# x86_64 需要 ≥4.18，aarch64 需要 ≥5.5
```

升级内核或使用满足要求的 Linux 发行版。

---

### Q：提示 "BTF is not available" 或 "vmlinux not found"

**原因**：当前内核未启用 `CONFIG_DEBUG_INFO_BTF`，不支持 CO-RE 模式。

**解决方案**：
1. 检查 BTF 支持：
   ```bash
   ls /sys/kernel/btf/vmlinux
   ```
2. 如果不存在，使用非 CO-RE 版本：
   - 下载 `noncore` 后缀的预编译包
   - 或从源码编译：`make nocore`

---

### Q：提示 "the current user does not have CAP_BPF"

**原因**：当前用户权限不足。

**解决**：
```bash
# 方法 1：使用 sudo
sudo ecapture tls

# 方法 2：设置 Capabilities
sudo setcap 'cap_bpf,cap_perfmon,cap_sys_ptrace=eip' /usr/local/bin/ecapture
```

详见 [最小权限配置](最小权限配置)。

---

## 运行相关

### Q：eCapture 启动后无输出

**可能原因**：
1. 目标程序未使用 eCapture 当前探针对应的 TLS 库
2. 目标程序没有 TLS 流量

**排查步骤**：
```bash
# 1. 确认目标程序的 TLS 库
ldd $(which curl) | grep -E "ssl|gnutls|nss"

# 2. 根据结果选择正确的探针
# libssl.so → sudo ecapture tls
# libgnutls.so → sudo ecapture gnutls
# libnspr4.so → sudo ecapture nss

# 3. 使用 --debug 查看详细日志
sudo ecapture tls --debug
```

---

### Q：提示 "symbol not found" 或探针挂载失败

**原因**：找不到目标函数符号，可能是库路径错误或符号表被 strip。

**解决**：
```bash
# 1. 手动指定库路径
sudo ecapture tls --libssl=/path/to/libssl.so.3

# 2. 检查符号表
nm -D /usr/lib/x86_64-linux-gnu/libssl.so.3 | grep SSL_read

# 3. 对于 Go 程序，确保未 strip 符号表
file /path/to/go_binary  # 应显示 "not stripped"
```

---

### Q：提示 "lost N events"

**原因**：perf buffer 溢出，用户态处理速度跟不上内核态事件产生速度。

**解决**：
1. 使用 `--pid` 限制捕获范围
2. 使用 `keylog` 模式（数据量更小）
3. 增大 perf buffer：`--per_cpu_map_size`

---

### Q：库路径错误怎么办

**排查**：
```bash
# 查找系统中的 OpenSSL 库
find / -name "libssl.so*" 2>/dev/null

# 查找 GnuTLS 库
find / -name "libgnutls.so*" 2>/dev/null

# 查看动态链接器搜索路径
cat /etc/ld.so.conf
ldconfig -p | grep ssl
```

## Android 相关

### Q：Android 上运行报错

**常见问题**：
1. 需要使用 `ANDROID=1 make all` 编译的版本
2. BoringSSL 路径需要手动指定：
   ```bash
   sudo ecapture tls --libssl=/apex/com.android.conscrypt/lib64/libssl.so --ssl_version="boringssl 1.1.1"
   ```
3. `bash`、`mysqld`、`postgres` 模块在 Android 上**不可用**

---

## Docker 相关

### Q：Docker 容器内运行失败

**确认清单**：
```bash
# 1. 确保使用了足够的权限
docker run --rm \
  --cap-add=BPF --cap-add=PERFMON --cap-add=NET_ADMIN --cap-add=SYS_PTRACE \
  --pid=host --net=host \
  -v /sys/kernel/debug:/sys/kernel/debug:ro \
  -v /sys/fs/bpf:/sys/fs/bpf \
  gojue/ecapture:latest tls

# 2. 确认挂载了必要的文件系统
# /sys/kernel/debug（debugfs）和 /sys/fs/bpf（bpffs）
```

## 收集诊断信息

提交 Issue 时，请附上以下信息：

```bash
# 系统信息
uname -a
cat /etc/os-release

# BTF 支持
ls -la /sys/kernel/btf/vmlinux 2>&1

# eCapture 版本
ecapture --version

# 调试日志
sudo ecapture tls --debug 2>&1 | head -50

# OpenSSL 版本
openssl version
ldconfig -p | grep ssl
```
