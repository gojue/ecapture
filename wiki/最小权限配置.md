# 最小权限配置

eCapture 需要提升的权限来加载 eBPF 程序和挂载 uprobe。本页说明在不同内核版本下所需的最低 Linux Capabilities。

## 权限矩阵

### 内核 ≥5.8（推荐）

从 Linux 5.8 起，BPF 相关权限从 `CAP_SYS_ADMIN` 中拆分：

| Capability | 用途 |
|------------|------|
| `CAP_BPF` | 加载和管理 eBPF 程序 |
| `CAP_PERFMON` | 创建 perf 事件和读取 perf buffer（eBPF 输出使用） |
| `CAP_SYS_PTRACE` | 访问其他进程的内存映射（读取 `/proc/<pid>/maps`） |
| `CAP_NET_ADMIN` | TC（Traffic Control）挂载，pcapng 模式需要 |

### 内核 <5.8

| Capability | 用途 |
|------------|------|
| `CAP_SYS_ADMIN` | 涵盖 BPF 和 perf 权限 |
| `CAP_NET_ADMIN` | pcapng 模式需要 |

### 按模式分类

| eCapture 模式 | 内核 ≥5.8 | 内核 <5.8 |
|---------------|-----------|-----------|
| `text` | `CAP_BPF` + `CAP_PERFMON` + `CAP_SYS_PTRACE` | `CAP_SYS_ADMIN` |
| `keylog` | `CAP_BPF` + `CAP_PERFMON` + `CAP_SYS_PTRACE` | `CAP_SYS_ADMIN` |
| `pcapng` | `CAP_BPF` + `CAP_PERFMON` + `CAP_NET_ADMIN` + `CAP_SYS_PTRACE` | `CAP_SYS_ADMIN` + `CAP_NET_ADMIN` |

## 配置方法

### 方法 1：使用 sudo（最简单）

```bash
sudo ecapture tls
```

### 方法 2：使用 setcap（推荐用于重复使用）

```bash
# 内核 ≥5.8，text/keylog 模式
sudo setcap 'cap_bpf,cap_perfmon,cap_sys_ptrace=eip' /usr/local/bin/ecapture

# 内核 ≥5.8，pcapng 模式（额外需要 cap_net_admin）
sudo setcap 'cap_bpf,cap_perfmon,cap_net_admin,cap_sys_ptrace=eip' /usr/local/bin/ecapture

# 内核 <5.8
sudo setcap 'cap_sys_admin,cap_net_admin,cap_sys_ptrace=eip' /usr/local/bin/ecapture
```

设置后可以免 sudo 运行：
```bash
ecapture tls
```

> **注意**：`setcap` 权限存储在文件扩展属性中。更新二进制文件后需要重新设置。

验证权限：
```bash
getcap /usr/local/bin/ecapture
```

### 方法 3：Docker 细粒度 Capabilities

替代 `--privileged=true`，使用特定 Capabilities：

```bash
# 内核 ≥5.8
docker run --rm \
  --cap-add=BPF \
  --cap-add=PERFMON \
  --cap-add=NET_ADMIN \
  --cap-add=SYS_PTRACE \
  --pid=host \
  --net=host \
  -v /sys/kernel/debug:/sys/kernel/debug:ro \
  -v /sys/fs/bpf:/sys/fs/bpf \
  gojue/ecapture:latest tls

# 内核 <5.8
docker run --rm \
  --cap-add=SYS_ADMIN \
  --cap-add=NET_ADMIN \
  --cap-add=SYS_PTRACE \
  --pid=host \
  --net=host \
  -v /sys/kernel/debug:/sys/kernel/debug:ro \
  -v /sys/fs/bpf:/sys/fs/bpf \
  gojue/ecapture:latest tls
```

### Docker 必要挂载

| 挂载路径 | 权限 | 用途 |
|----------|------|------|
| `/sys/kernel/debug` | 只读 | 访问 debugfs 以挂载 uprobe |
| `/sys/fs/bpf` | 读写 | BPF 文件系统，用于 pin maps |

### Docker 必要参数

| 参数 | 用途 |
|------|------|
| `--pid=host` | 访问宿主机进程命名空间 |
| `--net=host` | 访问宿主机网络命名空间（pcapng 模式需要） |

## 运行时权限检测

eCapture 在启动时会自动检测权限（见 `cli/cmd/env_detection.go`）：

1. **内核版本检查**：验证最低内核版本
2. **Capability 检查**：验证 `CAP_BPF`（内核 ≥5.8）或 `CAP_SYS_ADMIN`（内核 <5.8）

权限不足时的错误提示：
```
the current user does not have CAP_BPF to load bpf programs.
Please run as root or use sudo or add the --privileged=true flag for Docker
```

## 安全最佳实践

1. **最小权限原则**：使用 `setcap` 或 Docker `--cap-add` 替代 root 运行
2. **限制范围**：使用 `--pid` 参数限定目标进程
3. **审计使用记录**：记录 eCapture 的部署时间、目的和操作人员
4. **用后即删**：审计完成后移除 Capabilities 或卸载 eCapture
5. **文件权限**：限制 eCapture 二进制文件的访问权限
   ```bash
   sudo chown root:security-audit /usr/local/bin/ecapture
   sudo chmod 750 /usr/local/bin/ecapture
   ```
