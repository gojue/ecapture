# 数据库流量捕获（MySQL / PostgreSQL）

## 概述

eCapture 提供 `mysqld` 和 `postgres` 两个探针模块，通过 uprobe 挂钩数据库服务器内部的查询分发函数，直接捕获 SQL 查询语句。适用于数据库安全审计场景。

---

## MySQL / MariaDB 审计

### 支持版本

- MySQL 5.6 / 5.7 / 8.0
- MariaDB（兼容 MySQL 协议的版本）

### 工作原理

通过 uprobe 挂钩 mysqld 进程的 `dispatch_command` 函数，该函数是 MySQL 处理所有客户端命令的入口点，包括 SQL 查询。

### 使用方法

```bash
# 自动发现 mysqld 进程
sudo ecapture mysqld

# 指定 mysqld 路径
sudo ecapture mysqld --mysqld=/usr/sbin/mysqld

# 指定目标 mysqld 进程
sudo ecapture mysqld --pid=12345
```

### 手动指定函数名或偏移量

当 mysqld 的符号表被 strip 或函数名不在符号表中时，可以手动指定：

```bash
# 使用 objdump 查找函数偏移
objdump -t /usr/sbin/mysqld | grep dispatch_command

# 手动指定偏移量
sudo ecapture mysqld --funcname=dispatch_command
```

### 输出格式

```
PID: 12345 | UID: 0 | Command: Query | SQL: SELECT * FROM users WHERE id = 1
PID: 12345 | UID: 0 | Command: Query | SQL: INSERT INTO logs (msg) VALUES ('test')
```

## PostgreSQL 审计

### 支持版本

- PostgreSQL 10 及以上版本

### 工作原理

通过 uprobe 挂钩 postgres 进程的 `exec_simple_query` 函数，捕获所有简单查询语句。

### 使用方法

```bash
# 自动发现 postgres 进程
sudo ecapture postgres

# 指定 postgres 路径
sudo ecapture postgres --postgres=/usr/lib/postgresql/14/bin/postgres

# 指定目标进程
sudo ecapture postgres --pid=12345
```

## 生产环境使用注意事项

### 性能影响

1. **uprobe 开销**：每次 SQL 查询执行时会触发 uprobe，增加约 1-2μs 延迟
2. **高并发场景**：在高 QPS 数据库上使用时，建议通过 `--pid` 限定目标进程
3. **perf buffer**：大量 SQL 事件可能导致 perf buffer 溢出

### 权限要求

- 需要 ROOT 权限或 `CAP_BPF` + `CAP_PERFMON` + `CAP_SYS_PTRACE`
- 不需要数据库层面的账号权限

### 安全建议

1. 捕获的 SQL 可能包含敏感数据（密码、个人信息），妥善保管输出文件
2. 仅在授权的审计场景中使用
3. 审计完成后及时停止 eCapture
