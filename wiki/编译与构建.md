# 编译与构建

## 前提条件

编译 eCapture **必须在 Linux 系统上进行**（物理机、虚拟机或 SSH 远程服务器）。不支持在 Windows 或 macOS 上编译。

### 工具链要求

| 工具 | 版本要求 |
|------|---------|
| Go | ≥1.24 |
| Clang | ≥9（推荐 ≥14） |
| LLVM (llc) | ≥9 |
| cmake | ≥3.18.4 |
| bpftool | 最新版 |
| pkgconf | - |
| libelf-dev | - |
| linux-tools-common | - |
| linux-tools-generic | - |

### 一键初始化（Ubuntu 20.04+）

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/gojue/ecapture/master/builder/init_env.sh)"
```

## 构建方式

### CO-RE 构建（推荐）

依赖 BTF 支持，编译产物可跨内核版本运行：

```bash
git clone --recurse-submodules https://github.com/gojue/ecapture.git
cd ecapture
make          # 完整构建：eBPF（CO-RE + noncore）→ assets → binary
```

等价于执行：`ebpf` → `ebpf_noncore` → `assets` → `build`

### 非 CO-RE 构建

不依赖 BTF，但编译产物仅适用于编译时的内核版本：

```bash
git clone https://github.com/gojue/ecapture.git
cd ecapture
make nocore   # 仅非 CO-RE：ebpf_noncore → assets_noncore → build_noncore
```

### 仅构建 Go 二进制

当 eBPF 字节码和 assets 已经就绪时：

```bash
make build    # 仅编译 Go 二进制
```

## 交叉编译

### arm64 交叉编译（在 amd64 主机上）

```bash
# 安装交叉编译工具链
sudo apt-get install gcc-aarch64-linux-gnu

# 交叉编译
CROSS_ARCH=arm64 make
```

### amd64 交叉编译（在 arm64 主机上）

```bash
sudo apt-get install gcc-x86-64-linux-gnu
CROSS_ARCH=amd64 make
```

### 内核头文件

交叉编译需要目标架构的内核头文件：

```bash
kernel_ver=$(uname -r | cut -d'-' -f 1)
sudo apt-get install -y linux-source-$kernel_ver
cd /usr/src
source_file=$(find . -maxdepth 1 -name "*linux-source*.tar.bz2")
source_dir=$(echo "$source_file" | sed 's/\.tar\.bz2//g')
sudo tar -xf $source_file
cd $source_dir
test -f .config || yes "" | sudo make oldconfig
```

## Android 构建

```bash
ANDROID=1 make all
```

Docker 构建环境见 `builder/Dockerfile`。

## Makefile 常用 Target

| Target | 说明 |
|--------|------|
| `make` / `make all` | 完整构建（CO-RE + 非 CO-RE） |
| `make nocore` | 非 CO-RE 构建 |
| `make build` | 仅 Go 二进制（需要 assets 就绪） |
| `make clean` | 清理 `bin/`、`bytecode/*.o`、`assets/ebpf_probe.go` |
| `make env` | 显示所有构建变量和工具版本 |
| `make format` | 格式化代码（C: clang-format, Go: gofmt） |
| `make test-race` | 运行单元测试（`go test -race`） |
| `make e2e` | 运行所有 E2E 测试 |

## 构建标签（Build Tags）

| 标签 | 说明 |
|------|------|
| `ebpfassets` | 使用嵌入的 eBPF 字节码（生产构建） |
| `!ebpfassets` | 使用存根文件（开发/IDE 环境） |
| `linux` | Linux 平台 |
| `netgo` | 纯 Go 网络实现 |
| `dynamic` | 动态链接模式 |

生产构建命令使用：`-tags 'linux,netgo,ebpfassets,dynamic'`

## 关键构建文件

| 文件 | 说明 |
|------|------|
| `Makefile` | 主构建文件 |
| `variables.mk` | 构建变量、目标架构、eBPF 编译参数 |
| `functions.mk` | 构建辅助函数 |
| `builder/init_env.sh` | 环境初始化脚本 |
| `builder/Dockerfile` | Docker 构建环境 |
| `builder/Makefile.release` | 发布构建（RPM/DEB/tar.gz） |

## 中国用户加速

在 `make` 编译前设置 Go 代理：

```bash
export GOPROXY=https://goproxy.cn
```

## libpcap 说明

eCapture 静态链接 `libpcap`（位于 `lib/libpcap/`），通过 CGO 链接。如果系统中没有 `/usr/local/lib/libpcap.a`，`make` 命令会自动编译并安装 libpcap。

> **注意**：如果系统已安装了 libpcap 但没有静态库文件，`make` 可能会覆盖系统的 libpcap 头文件。
