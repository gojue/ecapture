# 支持平台与版本说明

## 操作系统与架构

| 平台 | 架构 | 最低内核版本 | 说明 |
|------|------|------------|------|
| Linux | x86_64 | 4.18 | 主流支持平台 |
| Linux | aarch64 (arm64) | 5.5 | 完整功能支持 |
| Android | aarch64 (GKI) | 5.5 | 支持 BoringSSL 捕获，部分模块不可用 |

> **不支持**：Windows、macOS。eCapture 依赖 Linux 内核的 eBPF 子系统，仅能在 Linux 内核环境下运行。

## CO-RE 与非 CO-RE 模式

eCapture 支持两种 eBPF 字节码加载模式：

### CO-RE 模式（Compile Once – Run Everywhere）

- **推荐模式**：利用 BTF（BPF Type Format）信息实现跨内核版本的可移植性
- **前提条件**：内核启用 `CONFIG_DEBUG_INFO_BTF=y`
- **检测方法**：
  ```bash
  ls /sys/kernel/btf/vmlinux
  ```
  如果文件存在，说明支持 CO-RE 模式。
- **优势**：无需为每个内核版本单独编译，一份字节码适用于所有支持 BTF 的内核

### 非 CO-RE 模式（Non-Core）

- **兼容模式**：针对不支持 BTF 的旧内核（如早期 CentOS 7、Ubuntu 18.04 等）
- **编译方式**：使用 `make nocore` 编译，需要本地内核头文件
- **适用场景**：内核版本满足最低要求但未启用 BTF 的环境

### 选择建议

| 条件 | 建议模式 |
|------|---------|
| 内核支持 BTF（`/sys/kernel/btf/vmlinux` 存在） | CO-RE |
| 内核不支持 BTF 但版本 ≥4.18 | 非 CO-RE |
| 需要分发到多个不同内核版本的机器 | CO-RE（优先） |
| 旧版 CentOS / Ubuntu / Debian | 非 CO-RE |

## 已验证的发行版

以下发行版已经过测试验证：

- **Ubuntu** 20.04 / 22.04 / 24.04（推荐）
- **CentOS** 8 / Stream 8 / Stream 9
- **Debian** 11 / 12
- **Fedora** 36+
- **RHEL** 8 / 9
- **Android** 13 / 14 / 15 / 16（GKI 内核）

## Android 平台说明

Android 平台使用 GKI（Generic Kernel Image）内核，eCapture 通过交叉编译提供支持：

- 编译命令：`ANDROID=1 make all`
- 主要用于捕获 BoringSSL 加密通讯（Android 系统默认使用 BoringSSL）
- BoringSSL 版本分支：`a_13`（Android 13）、`a_14`、`a_15`、`a_16`、`na`（非 Android BoringSSL）
- **限制**：Android 平台**不支持** `bash`、`mysqld`、`postgres` 探针

## 版本功能差异

eCapture 各版本的核心功能差异参见 [CHANGELOG.md](https://github.com/gojue/ecapture/blob/master/CHANGELOG.md)。主要里程碑包括：

- **v0.1.x**：初始版本，支持 OpenSSL 明文捕获
- **v0.4.x**：新增 pcapng 模式、Keylog 模式
- **v0.7.x**：新增 GoTLS、GnuTLS、NSS 探针
- **v0.8.x**：新增 HTTP/2 解析、eCaptureQ WebSocket 推送
- **v2.0.x**：架构重构，引入 Domain 接口、工厂模式、事件分发器
