# 项目简介与核心能力

## 项目背景

eCapture 是一款基于 Linux eBPF 技术的安全工具，通过在用户空间动态链接库（如 OpenSSL、BoringSSL、GnuTLS、NSS）的关键函数上挂载 `uprobe`，在数据被加密之前（或解密之后）直接读取明文内容。与传统的中间人代理（mitmproxy）或抓包工具（tcpdump）不同，eCapture **无需安装 CA 证书、无需修改目标应用程序、无需代理转发**。

## 三大核心能力

### 1. TLS/SSL 明文捕获

- 支持 **OpenSSL**（1.0.x / 1.1.x / 3.x 及更新版本）、**BoringSSL**（含 Android 各版本分支）、**GnuTLS**（3.6.12 ~ 3.8.7）、**NSS/NSPR**
- 支持 **GoTLS**（Go 语言内置 `crypto/tls` 库）
- 三种输出模式：
  - `text` 模式：直接输出明文内容
  - `pcap`/`pcapng` 模式：与 Wireshark 联动，保存完整数据包
  - `keylog` 模式：保存 TLS 密钥（NSS Key Log 格式），供 Wireshark 解密

### 2. 数据库 SQL 审计

- **MySQL/MariaDB**：支持 5.6 / 5.7 / 8.0，通过 `dispatch_command` 函数 uprobe 捕获 SQL 查询
- **PostgreSQL**：支持 10+，通过 `exec_simple_query` 函数 uprobe 捕获查询语句

### 3. Shell 命令审计

- **Bash**：通过 `readline` 库函数 uprobe 捕获命令输入
- **Zsh**：通过 `zle_main` 相关函数 uprobe 捕获命令输入
- 输出包含用户名、PID、时间戳、命令内容，适用于主机安全审计和合规场景

## 与传统工具的对比

| 特性 | eCapture | tcpdump | mitmproxy |
|------|----------|---------|-----------|
| 需要 CA 证书 | ❌ 不需要 | ❌ 不需要（但无法解密） | ✅ 需要 |
| 可查看加密明文 | ✅ 是 | ❌ 否 | ✅ 是 |
| 需要代理设置 | ❌ 不需要 | ❌ 不需要 | ✅ 需要 |
| 侵入性 | 无侵入 | 无侵入 | 需配置代理 |
| 支持 HTTP/2 | ✅ 是 | ❌ 加密流量 | ✅ 是 |
| 支持非 HTTP 协议 | ✅ 原始 TLS 数据 | ✅ 原始数据包 | ❌ 仅 HTTP(S) |
| 运行平台 | Linux/Android | 跨平台 | 跨平台 |
| 所需权限 | ROOT 或特定 Capabilities | ROOT | 普通用户 |

## 典型使用场景

1. **安全审计**：对生产环境中的加密流量进行明文审查，无需修改应用配置
2. **流量分析**：结合 Wireshark 分析 HTTPS/HTTP2/QUIC 通讯细节
3. **数据库监控**：实时捕获 MySQL/PostgreSQL 查询语句，用于 SQL 审计
4. **合规审查**：记录 Shell 操作命令，满足等保/SOX 等合规要求
5. **渗透测试**：在授权测试中捕获目标系统的加密通讯内容
6. **故障排查**：快速定位 TLS 通讯异常，无需配置 SSLKEYLOGFILE

## 探针模块一览

| 模块 | CLI 子命令 | 捕获目标 | 实现目录 |
|------|-----------|---------|---------|
| OpenSSL/BoringSSL | `tls` | SSL_read/SSL_write | `internal/probe/openssl/` |
| GoTLS | `gotls` | crypto/tls.Read/Write | `internal/probe/gotls/` |
| GnuTLS | `gnutls` | gnutls_record_recv/send | `internal/probe/gnutls/` |
| NSS/NSPR | `nss` | PR_Read/PR_Write | `internal/probe/nspr/` |
| MySQL | `mysqld` | dispatch_command | `internal/probe/mysql/` |
| PostgreSQL | `postgres` | exec_simple_query | `internal/probe/postgres/` |
| Bash | `bash` | readline | `internal/probe/bash/` |
| Zsh | `zsh` | zle_main | `internal/probe/zsh/` |
